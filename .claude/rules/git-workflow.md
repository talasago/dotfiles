# Git 運用（コミット・ブランチ・プッシュ）

このルールは Claude Code on the web の実行環境向け。ローカルPCでは `.claude/settings.local.json`（git未追跡）の `claudeMdExcludes` で本ファイルを除外しているためロードされず、ローカルでのGit運用はユーザーの判断に委ねる。

## コミット分割の方針

- コミットは意図ごとに分割する（1コミット＝1変更理由）。大量差分を1コミットにまとめない。

## ブランチ運用

- PRマージ後は同じブランチを再利用しない
- PRマージ後の新しい作業は必ず最新の main から新しいブランチを切って開始する

## プッシュ後のコンフリクト確認

- `git push` 後は `git diff origin/main...HEAD --name-only` 等で main とのコンフリクト有無を自分で確認する。
  - **コンフリクトが明らかに main 側の追加を取り込むだけで解消できる場合**：`git rebase origin/main` で自力解消して再プッシュする
  - **どちらを採用すべきか少しでも迷う場合**：両方の内容を提示し、どちらを採用すべきかをユーザーに判断を仰いでから解消する
- ユーザーにコンフリクトを気づかせない・促させないことが原則。
